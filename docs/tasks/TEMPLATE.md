# 当前任务模板

此模板用于 `docs/tasks/current.md` 中已承诺任务。Feature Spec 描述完整用户能力，Current Task 描述其中一个可独立执行、验证和交接的工作单元。

```markdown
### 任务名称

- **状态**：待开始 | 进行中 | 已完成
- **开始日期**：YYYY-MM-DD（待开始时省略）
- **Feature Spec**：`docs/features/example.md`
- **目标**：本任务独立交付的一个主要用户行为，或必要且可独立验证的底层契约。
- **范围**：本任务必须完成的行为和必要失败保护。
- **非范围**：留给后续任务或明确不在本功能内处理的内容。
- **依赖**：开始本任务前必须完成的任务、规格或系统能力。
- **实施要点**：必要的实现顺序或边界；不写可直接从代码恢复的流水账。
- **完成标准**：可观察结果及本任务实际需要执行的自动化、构建或人工验证。
- **当前进度**：仅进行中任务填写，记录真实完成边界和下一步。
- **验证记录**：完成时填写实际执行结果；未运行的检查不得写为通过。
- **交接信息**：仅记录下一会话无法轻易从代码与 Git 恢复的信息。
```

拆分判断：如果一个任务包含多个可分别演示的用户流程、多个独立破坏性选择，或其中一部分完成后已经可以单独验收，应继续拆分。跨越前端、IPC、Rust 核心和测试不自动意味着任务过大；若这些修改共同交付同一个行为，应保持为一个垂直切片。
