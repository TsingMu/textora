- **背景**：项目需要让个人开发与多种 AI 编程工具共享稳定意图和当前状态，同时避免文档负担。
- **决策**：按 `AGENTS.md` 定义的职责维护产品、架构、决策、功能规格、当前任务和 backlog；只记录无法轻易从代码与 Git 历史恢复的信息。
- **影响**：功能从 backlog 进入实现前应形成必要规格；完成实现时同步验证结果和当前任务状态，但不为流程机械创建内容。

## D-002：产品形态与基础内容范围

- **状态**：部分已取代（平台范围由 D-003 取代）
- **日期**：2026-07-14
- **背景**：需要明确项目最基础的产品边界。
- **决策**：Textora 是 macOS 桌面文本编辑器，支持范围包含普通文本、代码、Markdown 和 Mermaid。
- **影响**：具体支持深度由后续功能规格决定；本决策不预设 UI 框架、编辑器内核、渲染技术、最低系统版本或分发方式。

## D-003：采用 macOS 与 Windows 双平台产品范围

- **状态**：已接受
- **日期**：2026-07-14
- **背景**：Textora 的目标是通用桌面文本编辑器，不局限于 macOS。
- **决策**：首版支持 macOS 13+（arm64/x86_64）与 Windows 10 22H2+ / Windows 11（x64）；Windows 7 与 Windows ARM64 不在首版范围。
- **影响**：产品与测试必须维持两平台行为一致；不能采用仅服务单个平台的主 UI 技术栈。

## D-004：采用 Tauri、React、CodeMirror 与 Rust

- **状态**：已接受
- **日期**：2026-07-14
- **背景**：产品同时需要跨平台桌面能力、成熟文本编辑体验和 Markdown/Mermaid Web 生态，并希望避免为每个应用捆绑完整浏览器。
- **决策**：使用 Tauri 2 作为桌面壳，React + TypeScript + Vite 构建界面，CodeMirror 6 作为编辑器，Rust 负责文件与编码安全边界。
- **影响**：macOS WebKit 与 Windows WebView2 必须分别验证；文件系统通过受限 Tauri 命令暴露，前端不获得宽泛权限。

## D-005：首版文件规模与编码边界

- **状态**：已接受
- **日期**：2026-07-14
- **背景**：基础编辑必须先明确可验证的文件规模与无损编码规则。
- **决策**：首版支持最大 50 MiB 的 UTF-8、UTF-8 BOM 与严格 GBK/Windows CP936 文件。拒绝 GB18030 四字节序列和无法无损识别的编码。GBK 无法表示编辑内容时阻止保存并提供另存为 UTF-8。
- **影响**：Rust 文档核心负责无替换解码/编码、换行元数据、冲突检测和原子保存；不得静默改变编码或插入替代字符。
