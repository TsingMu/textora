# 基础文本编辑

> 状态：已确认

## 背景与目标

交付 Textora 的首个可运行跨平台切片：用户能在 macOS 与 Windows 上安全地新建、打开、编辑和保存 UTF-8 或 GBK 文本文件。

## 范围

- 新建文档，打开不超过 50 MiB 的本地文本文件。
- 使用 CodeMirror 编辑、选择、撤销和重做。
- 保存、另存为、未保存关闭保护和多标签会话。
- UTF-8、UTF-8 BOM、严格 GBK/CP936 的检测、显示、重新打开与显式转换。
- LF、CRLF 与混合换行识别；普通保存保持单一原换行风格。
- 外部修改检测、保存冲突提示和原子保存。
- 状态栏显示编码、换行与修改状态。

## 非范围

- GB18030、其他传统编码、超过 50 MiB 文件、十六进制模式。
- 代码语言服务、Markdown/Mermaid 预览、插件、协作和自动更新。

## 用户流程

1. 用户新建文档或通过系统对话框选择文件。
2. 应用读取一致文件快照，严格识别编码并显示内容。
3. 用户编辑内容；标签和状态栏显示未保存状态。
4. 用户保存到原路径或选择编码后另存为。
5. 若文件被外部修改、换行混合或 GBK 无法表示内容，应用在写入前要求用户处理。

## 行为规则与边界情况

- 自动检测顺序为 UTF-8 BOM、严格 UTF-8、严格 GBK；纯 ASCII 按 UTF-8。
- GBK 校验拒绝四字节 GB18030 序列；解码和编码都不插入替代字符。
- GBK 不可编码字符必须阻止保存并报告字符位置，允许另存为 UTF-8。
- 新文件默认 UTF-8、无 BOM、LF。
- 混合换行首次保存前必须显式选择 LF 或 CRLF。
- 发生外部修改时提供重新加载、覆盖或取消。
- 打开、读取、编码或保存失败不能丢失当前编辑内容，也不能清除未保存状态。
- 关闭未保存标签或窗口时提供保存、放弃或取消。

## 验收条件

- [ ] macOS 13+ 与 Windows 10 22H2+ 均可构建并启动应用。
- [ ] 可新建、打开、编辑、撤销/重做、保存及另存文本文件。
- [ ] UTF-8、UTF-8 BOM、ASCII 与合法 GBK 文件被正确识别且普通保存保持编码。
- [ ] 非法 GBK、GB18030 四字节和未知编码被拒绝且原文件不变。
- [ ] GBK 不可编码字符不会被替换或写入，并可另存为 UTF-8。
- [ ] LF/CRLF 被保持；混合换行未经选择不能覆盖保存。
- [ ] 50 MiB 允许打开，超过限制时明确拒绝。
- [ ] 外部修改、保存失败与关闭未保存文档均有保护。
- [ ] 前端不具有宽泛文件系统、shell 或网络权限。

## 依赖与约束

- 遵守 `docs/ARCHITECTURE.md` 的文档权威源、二进制 IPC 和文件安全不变量。
- 技术方案由 D-003、D-004 与 D-005 约束。
